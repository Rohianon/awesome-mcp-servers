# Contributing

Thank you for considering contributing! This list is community-driven and thrives on quality contributions.

## Guidelines

### Adding an Entry

1. **Fork** this repository.
2. **Create a branch**: `git checkout -b add-tool-name`
3. **Add your entry** in the appropriate category in `README.md`.
4. **Submit a Pull Request** with a clear description of what you're adding and why.

### Entry Format

Each entry should follow this format:

```markdown
- [Name](https://url.com/) - A concise description ending with a period.
```

- Description should be **one sentence** and explain what makes the tool useful.
- Start with a capital letter, end with a period.
- No promotional language ("best", "amazing", "revolutionary").

### Quality Standards

Entries must meet these criteria:

- **Actively maintained** — last commit or release within the past 12 months.
- **Documented** — has a README or docs explaining usage.
- **Relevant** — clearly fits the category it's placed in.
- **Not a duplicate** — check that a similar tool isn't already listed.

### What NOT to Add

- Paid tools with no free tier or trial (unless they're industry-standard).
- Abandoned or archived projects.
- Self-promotional content without substance.
- Tools that don't relate to the list's topic.

### Suggesting New Categories

Open an issue first to discuss new categories before adding them.

## Code of Conduct

By participating, you agree to abide by our [Code of Conduct](code-of-conduct.md).
